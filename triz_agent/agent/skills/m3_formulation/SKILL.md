# 问题定型

## 任务
基于根因分析结果，判断矛盾类型，提取标准化的 TRIZ 矛盾对。

## 输入
你会收到用户问题和之前的分析结果（包括根因分析的结果）。基于这些信息判断矛盾类型并提取矛盾对。

## 矛盾类型判断

### 技术矛盾（Tech Contradiction）
**特征**：改善参数 A 导致参数 B 恶化（两个不同参数之间的 trade-off）

**示例**：
- 改善"结构强度" → 恶化"建造成本"
- 改善"墨水供应量" → 恶化"纸张清洁度"

**处理方法**：提取改善方面和恶化方面（两个不同参数），查矛盾矩阵获得发明原理。

### 物理矛盾（Physical Contradiction）
**特征**：同一参数需要同时满足两个相反的状态（如"既要大又要小"）

**示例**：
- 墨水流量：书写时要"大"，停笔时要"小"
- 笔尖硬度：书写时要"硬"，携带时要"软"

**处理方法**：提取矛盾参数、两个相反状态、选择分离类型，查分离原理获得发明原理。

## 物理矛盾分析步骤

1. **提取矛盾参数**：同一参数（如"墨水流量"、"供给速率"、"重量"、"强度"）
2. **判断两个相反状态**：如"大"vs"小"、"硬"vs"软"、"热"vs"冷"
3. **选择分离类型**：
   - **空间分离（Space）**：在不同空间位置解决 → 关键词：局部/整体、内部/外部、上面/下面
   - **时间分离（Time）**：在不同时间段解决 → 关键词：工作时/闲置时、白天/夜晚、书写时/停笔时
   - **条件分离（Condition）**：在不同条件下解决 → 关键词：高温时/低温时、高压时/低压时、速度快时/慢时
   - **系统分离（System）**：在系统层级解决 → 关键词：组件级/系统级、子系统/超系统

## 输出格式

请输出以下 Markdown 内容：

### 矛盾类型

（技术矛盾 / 物理矛盾）

### 矛盾对

（技术矛盾）
- **改善方面**：（参数A，如"结构强度"）
- **恶化方面**：（参数B，如"建造成本"）

（物理矛盾）
- **矛盾参数**：（同一参数，如"墨水流量"）
- **状态1**：（如"大流量"）
- **状态2**：（如"小流量"）
- **分离类型**：（空间/时间/条件/系统）

### 矛盾描述

（一句话描述，如"改善结构强度导致建造成本上升"或"书写时需要大流量，停笔时需要小流量"）

### 依据

（简要说明判断依据）

## 示例

**技术矛盾示例：**
```
### 矛盾类型
技术矛盾

### 矛盾对
- **改善方面**：结构强度
- **恶化方面**：建造成本

### 矛盾描述
提高建筑物抗震能力需要增强结构强度，但会导致建造成本大幅上升

### 依据
根据根因分析，关键问题是"增强结构强度导致成本上升"，属于两个不同参数之间的 trade-off。
```

**物理矛盾示例：**
```
### 矛盾类型
物理矛盾

### 矛盾对
- **矛盾参数**：墨水流量
- **状态1**：大流量（书写时）
- **状态2**：小流量（停笔时）
- **分离类型**：时间

### 矛盾描述
书写时需要大墨水流量以保证流畅度，停笔时需要小流量以避免污染纸张

### 依据
根据根因分析，关键问题是"同一墨水流量参数在书写和停笔时需要相反的状态"，属于物理矛盾，可用时间分离解决。
```

## Gotchas（常见陷阱）

1. **矛盾类型误判**：LLM 倾向于把所有问题都标为技术矛盾 → 如果描述的是"同一参数的两个对立需求"，应该是物理矛盾
2. **物理矛盾格式错误**：物理矛盾应输出"矛盾参数+状态1+状态2+分离类型"，而非两个不同参数
3. **属性描述过长**：改善方面/恶化方面/矛盾参数应是2-6个中文词的简洁描述
4. **缺少根因上下文**：如果之前的分析中根因参数和关键问题为空，应要求补充信息

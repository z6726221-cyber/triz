# M3 问题定型

## 任务
基于根因分析结果，提取标准化的 TRIZ 矛盾对（技术矛盾或物理矛盾）。

## 输入
你会收到用户问题和之前的分析结果（包括 M2 的根因分析）。基于这些信息判断矛盾类型并提取矛盾对。

## 分析步骤

1. 阅读根因参数和关键问题
2. 判断矛盾类型：
   - 如果描述的是"改善 A 导致 B 恶化" → 技术矛盾
   - 如果描述的是"既要 X 又要 Y（对立需求）" → 物理矛盾
3. 提取矛盾对：
   - **技术矛盾**：改善方面 = 需要改善的核心属性，恶化方面 = 随之恶化的核心属性
   - **物理矛盾**：改善方面 = 一个对立状态，恶化方面 = 另一个对立状态

## 输出格式

请输出以下 Markdown 内容：

### 矛盾类型

（技术矛盾 / 物理矛盾）

### 矛盾对

- **改善方面**：（2-6个中文词的简洁描述）
- **恶化方面**：（2-6个中文词的简洁描述）

### 矛盾描述

（一句话描述矛盾关系，如"改善结构强度导致建造成本上升"）

### 依据

（简要说明判断依据）

## 示例

**技术矛盾示例：**
- 改善方面：结构强度
- 恶化方面：建造成本
- 矛盾描述：提高建筑物抗震能力需要增强结构强度，但会导致建造成本大幅上升

**物理矛盾示例：**
- 改善方面：大容量
- 恶化方面：轻薄
- 矛盾描述：既要电池容量大又要手机轻薄

## Gotchas（常见陷阱）

1. **矛盾类型误判**：LLM 倾向于把所有问题都标为技术矛盾 → 如果用户描述的是"既要 X 又要 Y"（对立需求），应该是物理矛盾
2. **属性描述过长**：改善方面/恶化方面应是 2-6 个中文词的简洁描述，而非完整句子
3. **物理矛盾格式错误**：物理矛盾的两个方面应是对立状态（如"大容量"vs"轻薄"），而非两个不同属性
4. **缺少根因上下文**：如果之前的分析中根因参数和关键问题为空，应要求补充信息

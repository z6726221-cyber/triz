# M1 功能建模 — 输出示例

## 示例 1：手机电池续航问题

**用户问题**："手机电池续航太短，一天要充两次电，怎么办？"

### SAO 三元组

| 主体 | 动作 | 客体 | 类型 |
|------|------|------|------|
| 电池 | 提供 | 电能 | useful |
| 电池 | 消耗 | 电能（自放电） | harmful |
| 用户 | 充电 | 电池 | excessive |
| 充电电路 | 转换 | 电能（交流→直流） | useful |
| 显示屏 | 消耗 | 电能 | harmful |

### 可用资源

- **物质**：锂离子电池、石墨烯、硅纳米线、电解液、铜箔、铝箔
- **场**：电场（充电电压）、磁场（无线充电）、热场（散热）、电磁场（屏蔽）
- **空间**：电池仓内部空间、手机外壳、电路板间隙、屏幕下方空间
- **时间**：充电时段（夜间）、待机时段、高峰使用时段、电池老化周期
- **信息**：电量显示数据、App 功耗排行、用户使用习惯、电池健康度
- **功能**：快充协议、省电模式、后台进程管理、屏幕亮度自适应

### 理想最终结果（IFR）

系统在无需用户干预的情况下，通过自身资源管理持续提供全天候电能供给，用户无需关心充电频率和电池状态，手机在任何使用强度下均能自动维持足够的电能储备。

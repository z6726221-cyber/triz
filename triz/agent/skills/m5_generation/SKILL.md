# M5 方案生成

## 任务
基于发明原理和上下文信息，生成具体可执行的方案草稿。

## 输入
你会收到用户问题和之前的分析结果（包括矛盾描述、发明原理、IFR、资源、跨界案例等）。基于这些信息生成方案。

## 流程

1. 阅读上下文中的发明原理和跨界案例（如有）
2. 基于原理和案例，结合用户具体场景，生成 2-3 个具体方案
3. 每个方案必须引用发明原理编号，描述至少 100 字

详细的生成指南，请参考 `references/generation_guide.md`。

## 输出格式

### 方案

#### 方案 1：（标题）

**应用原理**：原理 X（名称）

**方案描述**：（至少100字的具体技术方案描述）

**资源映射**：（使用了哪些现有资源）

#### 方案 2：（标题）

...

## 约束

1. 每个方案必须明确引用一个或多个发明原理编号
2. 方案必须具体、可执行，避免泛泛而谈（至少100字描述）
3. 优先使用用户已有的资源，避免引入全新组件
4. 如果上下文中有跨界案例，优先参考这些案例
5. 如果无法生成方案，明确说明原因

## Gotchas（常见陷阱）

1. **方案未引用原理**：LLM 生成的方案容易泛泛而谈 → 每个方案必须明确引用发明原理编号
2. **方案过于简短**：LLM 倾向于一句话方案 → 描述至少 100 字，包含具体的技术实现细节
3. **忽略已有资源**：方案应优先使用用户已有的资源（来自 M1 的资源盘点），避免引入全新组件
4. **重复已知原理**：方案只是复述原理本身而非具体应用 → 应结合用户场景做迁移

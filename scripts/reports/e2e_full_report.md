## 六、结论

**两种模式全部 35 个用例通过，功能正确性完全一致。**

- Agent 平均耗时是 Orchestrator 的 **2.0x**（正常用例差距更大，对抗/边界用例差距较小）
- Agent 在对抗性和边界输入上的处理速度与 Orchestrator 接近，说明 input_classifier 拦截机制在两种模式下均有效
- Agent 在正常工程问题上耗时更长，主要因为 ReAct 循环中每步都需要额外的 LLM 调用来决策下一步
- 所有 35 个用例均无失败或超时，系统鲁棒性良好
